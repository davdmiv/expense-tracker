## Требования

Приложение должно запускаться из командной строки и обладать следующими функциями:
* Пользователи могут добавить расходы, указав описание и сумму.
* Пользователи могут обновлять информацию о расходах.
* Пользователи могут удалить расходы.
* Пользователи могут просмотреть все расходы.
* Пользователи могут просмотреть сводку всех расходов.
* Пользователи могут просмотреть сводку расходов за конкретный месяц (текущего года).

Вот некоторые дополнительные функции, которые вы можете добавить в приложение:
* Добавьте категории расходов и дайте пользователям возможность фильтровать расходы по категориям.
* Предоставьте пользователям возможность устанавливать бюджет на каждый месяц и отображайте предупреждение, если пользователь превысит бюджет.
* Предоставьте пользователям возможность экспортировать расходы в CSV-файл.

Ниже приведён список команд и ожидаемый результат их выполнения:

```bash
$ expense-tracker add --description "Lunch" --amount 20
# Expense added successfully (ID: 1)

$ expense-tracker add --description "Dinner" --amount 10
# Expense added successfully (ID: 2)

$ expense-tracker list
# ID  Date       Description  Amount
# 1   2024-08-06  Lunch        $20
# 2   2024-08-06  Dinner       $10

$ expense-tracker summary
# Total expenses: $30

$ expense-tracker delete --id 2
# Expense deleted successfully

$ expense-tracker summary
# Total expenses: $20

$ expense-tracker summary --month 8
# Total expenses for August: $20
```

## Выполнение

Вы можете реализовать приложение, используя любой язык программирования на ваш выбор. Вот несколько предложений:

* Используйте любой язык программирования и любой доступный модуль для анализа аргументов команд (например, Python с помощью `<p>` argparse, Node.js с помощью `<p>` commanderи т. д.).

* Для хранения данных о расходах используйте простой текстовый файл. Вы можете использовать форматы JSON, CSV или любой другой.

* Добавьте обработку ошибок для работы с недопустимыми входными данными и особыми случаями (например, отрицательные суммы, несуществующие идентификаторы расходов и т. д.).

* Используйте функции для модульной организации кода, упрощения тестирования и сопровождения.
